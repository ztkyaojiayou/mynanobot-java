# C-013: 前端响应式改造 - 测试指南

## 🚀 启动应用

```bash
cd huazai-trip-front
npm run dev -- --host 0.0.0.0 --port 5173
```

访问：http://localhost:5173

## ✅ 测试步骤

### 1. 默认显示桌面端布局
- 首次访问时，页面应该显示**宽屏布局**（无手机外壳）
- 表单字段应该横向排列，最大宽度 1200px
- 不应该看到 PhoneFrame 手机模拟器

### 2. 切换至手机端
- 点击顶部导航栏中间的"设备模式"切换器
- 点击"手机端"按钮
- 页面应该变为**手机模拟器模式**
- 看到 360x660 的手机外壳、状态栏、底部 Tab

### 3. 切换回桌面端
- 再次点击"桌面端"按钮
- 页面恢复为宽屏布局

### 4. 持久化测试
- 在桌面端模式下刷新页面
- 应该保持桌面端布局
- 在手机端模式下刷新页面
- 应该保持手机模拟器模式

### 5. 小屏自动适配
- 将浏览器窗口缩小到 <768px 宽度
- 应该自动切换到手机模拟器模式
- 即使之前选择的是桌面端

## 🔍 调试技巧

### 检查 body 类名
打开浏览器开发者工具 Console，运行：
```javascript
document.body.className
// 期望输出："desktop-mode" 或 "mobile-mode"
```

### 检查 localStorage
```javascript
localStorage.getItem('device-mode')
// 期望输出："desktop" 或 "mobile"
```

### 检查当前路由
```javascript
router.currentRoute.value.path
// 期望输出：/intent, /planning/:id, 等
```

### 检查组件是否加载
在 DesktopLayout 中添加日志：
```typescript
onMounted(() => {
  console.log('DesktopLayout mounted')
  console.log('Current path:', router.currentRoute.value.path)
  console.log('Current component:', CurrentDesktopComponent.value)
})
```

## ❗ 常见问题

### Q: 页面仍然显示手机模拟器
**A:** 检查以下几点：
1. 确认 DesktopLayout.vue 已更新
2. 清除浏览器缓存后刷新
3. 检查 npm run dev 是否重新编译
4. 查看浏览器 Console 是否有错误

### Q: 切换后没有变化
**A:** 
1. 检查 DeviceSwitcher 组件是否正确调用 deviceStore.toggle()
2. 检查 localStorage 是否被浏览器扩展拦截
3. 尝试强制刷新（Ctrl+Shift+R）

### Q: 小屏不自动切换
**A:**
1. 检查 index.html 中的设备检测脚本
2. 确认 viewport meta 标签包含 `maximum-scale=1.0`
3. 检查 window.innerWidth 值
