# 数据模型与存储

> **作用域**: mynanobot-java 的数据模型和文件存储 Schema。
> **存储方式**: 纯文件存储（JSONL + JSON），不依赖数据库。

---

## 1. 存储分层

| 层 | 路径 | 格式 | 内容 | 生命周期 |
|----|------|------|------|---------|
| 会话历史 | `{workspace}/sessions/{key}/history.jsonl` | JSONL | 对话消息（每行一条 JSON） | 手动删除 |
| 会话元数据 | `{workspace}/sessions/{key}/metadata.json` | JSON | 会话名 (name) 等 | 手动删除 |
| 长期记忆 | `.nanobot/dream.json` | JSON | AI 提取的关键记忆条目 | 永久（LLM 管理） |
| 项目记忆 | `NANOBOT.md` | Markdown | AI 协作上下文（/init 生成） | 手动更新 |
| 任务追踪 | `.nanobot/workspace/tasks/{session}.json` | JSON | TaskStore 任务列表 | 会话级别 |
| 配置文件 | `config/config.yaml` | YAML | 模型、工具、MCP 配置 | 手动修改 |

---

## 2. 核心数据模型

### 2.1 会话消息（history.jsonl）

```json
{"role":"system","content":"你是 my-nanobot..."}
{"role":"user","content":"你好"}
{"role":"assistant","content":"你好！","tool_calls":[{"id":"1","type":"function","function":{"name":"read_file","arguments":"{\"path\":\".\"}"}}]}
{"role":"tool","tool_call_id":"1","content":"[TOOL_OK] ..."}
```

### 2.2 会话元数据（metadata.json）

```json
{"name":"调试 DeepSeek 流式问题"}
```

### 2.3 InboundMessage

| 字段 | 类型 | 说明 |
|------|------|------|
| channel | String | 来源通道（"cli", "api", "websocket"） |
| sessionId | String | 会话标识 |
| content | String | 用户文本内容 |
| metadata | Map | requestId, streamMode, useSearch, connectionId |
| timestamp | Instant | 消息时间 |
| media | List\<String\> | 附件路径 |

### 2.4 OutboundMessage

| 字段 | 类型 | 说明 |
|------|------|------|
| channel | String | 目标通道 |
| sessionId | String | 目标会话 |
| content | String | 响应内容 |
| metadata | Map | _stream_delta, _stream_end, _progress 等系统标记 |
| requestId | String | 用于 HTTP 同步匹配 |
| connectionId | String | 用于 WebSocket 路由 |

## 3. TaskStore 任务模型

```json
[
  {
    "id": "task-1",
    "subject": "实现用户注销功能",
    "description": "包括 JWT 黑名单和 Session 失效",
    "status": "completed",
    "blockedBy": [],
    "blocks": []
  }
]
```

状态流转: `pending → in_progress → completed`
