# 接口协议定义

> **作用域**: mynanobot-java 对外 REST API、CLI 命令和 MCP 协议约定。
> **版本**: v1.0.0

---

## 1. REST API（V2 Spring Boot 模式）

基础路径: `/api` · 媒体类型: `application/json` · 字符集: UTF-8

### 1.1 同步聊天

```
POST /api/chat
Content-Type: application/json

{
  "content": "你好",
  "sessionId": "chat-123",
  "useSearch": true
}
```

响应: `{ "content": "你好！我是 my-nanobot...", "sessionId": "chat-123" }`

机制: `MessageBus.waitForSessionResponse(sessionId, requestId, 120s)` 轮询匹配

### 1.2 流式聊天 (SSE)

```
POST /api/chat/stream
Content-Type: application/json

{ "content": "...", "sessionId": "...", "streamMode": true }
```

响应: `text/event-stream`，`SseEmitter` 实时推送每个 token

### 1.3 会话管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/sessions` | 列出所有会话（key, name, messageCount, lastModified） |
| GET | `/api/sessions/{key}` | 获取会话消息历史 |
| PATCH | `/api/sessions/{key}` | 重命名 `{"name":"新名称"}` |
| DELETE | `/api/sessions/{key}` | 删除会话及其文件 |

### 1.4 健康检查

```
GET /api/health → { "status": "UP" }
```

## 2. CLI 命令（V3 CLI 模式）

| 命令 | 说明 |
|------|------|
| `/help` | 列出所有命令 |
| `/init` | 分析项目生成 NANOBOT.md |
| `/plan` | 进入规划模式（只读分析出计划） |
| `/plan approve` | 审批计划，切换到执行模式 |
| `/mode <mode>` | 切换权限: plan / default / accept_edits / bypass |
| `/resume [key]` | 列出或恢复历史会话 |
| `/clear` | 清空当前上下文 |
| `/exit` | 退出 |

---

## 3. WebSocket（V2 Spring Boot 模式）

端点: `ws://host:port/ws`

消息格式:
```json
{ "type": "chat", "content": "...", "sessionId": "..." }
```

流式响应通过 `StreamResponseCallback.onStreamData()` → `session.getBasicRemote().sendText()`

---

## 4. MCP 协议

支持 stdio 和 streamableHttp 两种传输方式。工具注册后命名规则: `mcp_<server>_<tool>`。
