# 业务模型 — AI Agent 框架核心概念

> **作用域**: mynanobot-java 作为 AI Agent 框架的核心业务概念和实体关系。

---

## 1. 核心概念

```
用户输入（自然语言 / HTTP / WebSocket）
    │
    ▼
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│  Channel  │───▶│ MessageBus│───▶│ AgentLoop │───▶│ AgentRunner│
│ (入口)    │    │ (队列)    │    │ (状态机)  │    │ (LLM循环) │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
                                            │
                    ┌───────────────────────┼───────────────────────┐
                    ▼                       ▼                       ▼
            ┌───────────┐           ┌───────────┐           ┌───────────┐
            │   Tools   │           │ Providers │           │  Session  │
            │ (17 内置) │           │ (LLM API) │           │  Manager  │
            └───────────┘           └───────────┘           └───────────┘
```

### 核心实体

| 实体 | 说明 |
|------|------|
| **InboundMessage** | 入站消息（channel + sessionId + content + metadata） |
| **OutboundMessage** | 出站消息（流式标记: _stream_delta / _stream_end / _progress） |
| **TurnContext** | 会话上下文（消息历史、工具定义、token 统计、最终内容） |
| **TurnState** | 状态枚举（RESTORE→COMPACT→COMMAND→BUILD→RUN→SAVE→RESPOND→DONE） |
| **Tool** | 工具接口（name + description + parameters + isReadOnly + execute） |
| **LLMProvider** | 大模型接口（chat / chatStream） |
| **Command** | 命令接口（name + aliases + execute） |

---

## 2. 消息处理生命周期

```
RESTORE    加载会话历史 + 添加用户消息 → sessionManager.loadHistory()
COMPACT    检查 token 预算 → 超限时 Consolidator 压缩旧消息
COMMAND    检测 / 前缀 → 技能调用 或 内置命令（/stop /clear）
BUILD      组装 System Prompt（身份 + NANOBOT.md + PlanMode + Rules）
RUN        调用 LLM → 解析工具调用 → 执行工具 → 递归
SAVE       保存助手响应到会话历史
RESPOND    构建 OutboundMessage → 发布到 MessageBus
DONE       清理，等待下一消息
```

## 3. 三种运行模式

| 模式 | 入口 | 通道 | 适用场景 |
|------|------|------|---------|
| V3 CLI | `NanobotCliApplication` | 命令行终端 | 开发者日常使用 |
| V2 Web | `NanobotApplication` | HTTP/SSE/WebSocket | Web 服务 |
| V1 Standalone | `Nanobot.java` | 内嵌 HTTP Server | 学习参考 |

## 4. 权限模型

```
PLAN（只读） → DEFAULT（默认） → ACCEPT_EDITS（编辑放行） → BYPASS（全放行）
──────────────────────────────────────────────────────────────→ 信任度递增
```
