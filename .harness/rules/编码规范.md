# 编码规范

> **作用域**: mynanobot-java 全模块 Java 代码标准与约定。
> **基线**: Alibaba Java Coding Guidelines（阿里巴巴 Java 开发手册）— 推荐参考，不机械检查。
> **状态**: 强制（命名、异常处理，日志）；推荐（其他）

---

## 1. 命名约定

| 元素 | 规则 | 示例 |
|------|------|------|
| 类 / 接口 | 大驼峰 PascalCase | `AgentLoop`, `MessageBus`, `TurnContext` |
| 方法 / 变量 | 小驼峰 camelCase | `processMessage()`, `sessionId` |
| 常量 | 全大写 + 下划线 | `MAX_TOOL_RESULT_CHARS` |
| 包 | 全小写 | `com.nanobot.core.state` |
| Tool 类 | 以 `Tool` 结尾 | `ReadFileTool`, `ExecTool` |
| State 类 | 以 `State` 结尾 | `RestoreState`, `BuildState` |
| Provider 类 | 以 `Provider` 结尾 | `OpenAIProvider`, `DeepSeekProvider` |
| 测试类 | 被测类名 + `Test` | `MessageBusTest`, `ToolRegistryTest` |

---

## 2. Lombok 使用策略

| 注解 | 状态 | 说明 |
|------|------|------|
| `@Data` | ✅ 使用 | 本项目广泛使用（InboundMessage, OutboundMessage 等） |
| `@Getter` | ✅ 使用 | 只读场景 |
| `@AllArgsConstructor` | ✅ 使用 | 结合 @Data |
| `@Builder` | ✅ 使用 | 链式构建 |
| `@Slf4j` | ❌ 不用 | 本项目统一使用 `LoggerFactory.getLogger()` |

---

## 3. 日志约定

- 统一使用 `LoggerFactory.getLogger(getClass())`
- 级别：DEBUG(调试细节) / INFO(关键流程节点) / WARN(降级/重试) / ERROR(需人工介入)
- 中文日志 + 英文关键字段：`logger.info("Message processed in {}ms, tokens: {}", duration, tokens)`
- 禁止打印 API Key、Token、用户隐私

---

## 4. 异常处理

| 规则 | 说明 |
|------|------|
| 不吞异常 | 禁止空 catch；至少记录日志 |
| try-with-resources | 资源自动释放 |
| 异常信息 | 包含上下文，不泄露密钥 |
| 线程中断 | catch InterruptedException 时恢复 `Thread.currentThread().interrupt()` |

```java
// ❌ 禁止
try { call(); } catch (Exception e) {}

// ✅ 正确
try {
    messageBus.publishInbound(msg);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
    logger.warn("发送中断");
}
```

---

## 5. LLM / 外部调用约束

每一次 LLM 或外部 API 调用需具备：

| 维度 | 当前 |
|------|------|
| 超时 | AgentRunner: 工具 30s, LLM 300s |
| 重试 | executeToolWithRetry: 最多 3 次，间隔 1s |
| 结果截断 | 16,000 字符（防 token 爆炸） |
| 降级 | 连续 3 次工具失败 → 不带工具直接回答 |

---

## 6. 空安全

- public 方法入参用 `Objects.requireNonNull`
- 返回集合用空集合而非 `null`（`List.of()`, `Map.of()`）
- `Optional` 用于可能缺失的返回值
- 链式取值前判空

---

## 7. 并发约束

- AgentLoop: 单 daemon 线程消费入站消息（保证串行）
- 流式回调: `CopyOnWriteArrayList`（线程安全迭代）
- 会话锁: `ConcurrentHashMap + synchronized`（会话级串行）
- 工具执行: `ExecutorService` 线程池（并行执行同轮工具）

---

## 8. 安全红线

| 红线 | 规则 |
|------|------|
| 密钥 | API Key → 环墧变量或配置文件，禁止硬编码 |
| 日志 | 禁止打印 Token、密码、隐私信息 |
| 文件操作 | 限定在工作区范围内（PathGuard） |
| Shell 执行 | 默认需用户确认（CommandGuard + 交互确认） |

---

## 9. 提交规范

```
<type>: <简短描述>

type: feat | fix | refactor | test | docs | chore
```

示例:
```
feat: CLI 流式中断改为 Esc 键，引入 JLine 实现跨平台兼容
refactor: LLM Provider 创建抽取为 ProviderFactory 策略工厂
docs: 更新配置和编译章节匹配实际代码
```

**粒度**: 功能/bug 完成后 `mvn test` 通过再 `git commit`。
