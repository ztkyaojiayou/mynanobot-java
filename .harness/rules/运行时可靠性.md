# 运行时可靠性规范

> **作用域**: 约束 mynanobot-java 在运行时的健壮性——工具容错、消息不丢、会话持久、优雅关闭。
> **定位**: 单进程 Agent 的可靠性基线，不涉及分布式中间件。
> **状态**: 强制

---

## 1. Agent 调用可靠性

### 1.1 LLM 调用保护

| 机制 | 实现 |
|------|------|
| 超时 | HTTP 请求 300s |
| 重试 | `chatWithRetry` 默认 3 次（仅可重试错误） |
| 降级 | 连续 3 次工具失败 → `callLLMWithoutTools()` 强制不使用工具 |
| 迭代上限 | `maxToolIterations` (100) / `maxTurns` / `maxCost` 三重保护 |
| 会话取消 | `context.isCancelled()` 检查，支持 Esc 中断 |

### 1.2 工具执行保护

| 机制 | 实现 |
|------|------|
| 超时 | `executeToolWithRetry` 30s |
| 重试 | 最多 3 次，间隔 1s |
| 结果截断 | 16,000 字符防 token 爆炸 |
| 并行执行 | `CompletableFuture.allOf` 等待全部完成 |
| 权限检查 | `PermissionManager.check()` 四步管道 |

## 2. 消息总线可靠性

| 机制 | 实现 |
|------|------|
| 有界队列 | `ArrayBlockingQueue(100)` 防内存溢出 |
| 停止保护 | `running` 状态为 false 时静默丢弃消息 |
| 会话锁 | `ConcurrentHashMap + synchronized` 保证历史文件串行写入 |
| 增量保存 | `saveHistory()` 只追加新增消息，不全量覆写 |

---

## 3. 会话持久化可靠性

| 机制 | 实现 |
|------|------|
| 文件格式 | JSONL (history.jsonl) + JSON (metadata.json) |
| 原子写入 | try-with-resources + StandardOpenOption.APPEND |
| 损坏容错 | `loadHistory()` 逐行解析，单行异常跳过 |
| 会话隔离 | 每个 session 独立目录，并发锁保护 |

---

## 4. CLI 交互可靠性

| 机制 | 实现 |
|------|------|
| 流式中断 | JLine Terminal + Esc 检测（50ms 轮询） |
| 终端恢复 | JLine 自动恢复终端模式，异常时回退 Enter 中断 |
| 输入保护 | 单 Scanner 实例，避免多 Scanner 抢 stdin |
| 流式超时 | `sendMessage()` 等待最多 5 分钟 |

---

## 5. 优雅关闭

```java
Runtime.getRuntime().addShutdownHook(new Thread(() -> {
    agentLoop.stop();                        // 停止消费新消息
    messageBus.shutdown(5, TimeUnit.SECONDS); // 等待队列排空
    toolRegistry.shutdown();                  // 关闭工具线程池
    mcpManager.close();                       // 断开 MCP 连接
}));
```

## 6. 日志与可观测性

| 维度 | 实现 |
|------|------|
| 关键节点 | INFO: 消息入队/出队、LLM 调用、工具执行、响应发送 |
| 调试信息 | DEBUG: 状态流转、消息内容、token 统计 |
| 错误日志 | ERROR: 完整的异常栈 + 会话上下文 |
| 编码 | UTF-8（logback.xml 配置 `<charset>UTF-8</charset>`） |
