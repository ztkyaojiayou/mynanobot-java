# 工程结构规范

> **作用域**: 定义 mynanobot-java 的目录组织和包结构。单模块 Maven 项目，不涉及多模块依赖约束。
> **状态**: 强制

---

## 2. 包结构约定

- 一级包：按功能模块划分（`bus`, `core`, `tools`, `security`, `session`, ...）
- 接口和实现不强制分离，但 `providers/impl/` 和 `tools/impl/` 遵循此约定
- 禁止循环依赖：`core` → `tools` → `providers` 是单向依赖链
- 内部实现包（`impl/`、`state/`、`guard/`）仅被同模块引用

---

## 3. 文件组织约束（推荐）

| 约束 | 建议值 | 说明 |
|------|--------|------|
| 单文件行数 | ≤ 500 行 | AgentLoop 已从 973→579 行 |
| 单方法行数 | ≤ 50 行 | 提取小方法增加可读性 |
| 一个文件一个 public 类 | 强制 | Java 编译约束 |

---

## 4. 命名约定

| 元素 | 规则 | 示例 |
|------|------|------|
| 接口 | 语义化命名 | `Tool`, `LLMProvider`, `AgentState`, `Command` |
| 实现类 | 接口名 + Impl 或 Provider 后缀 | `OpenAIProvider`, `RestoreState` |
| 工具类 | 以 `Tool` 结尾 | `ReadFileTool`, `ExecTool`, `AskUserTool` |
| 测试类 | 被测类名 + `Test` | `MessageBusTest`, `ToolRegistryTest` |

---

## 5. 新增模块/功能约定

- 新工具 → `tools/impl/XxxTool.java` + 注册到 `BuiltinTools`
- 新命令 → `command/impl/XxxCommand.java` + 注册到 `CommandRegistry`
- 新 Provider → `providers/impl/XxxProvider.java` + 注册到 `ProviderFactory`
- 新状态处理器 → `core/state/XxxState.java` + 注册到 `AgentLoop.initStateHandlers()`
- 所有变更在 `.harness/changes/` 建档
