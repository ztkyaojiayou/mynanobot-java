# 工程结构规范

> **作用域**: 定义 mynanobot-java 的目录组织、模块划分、依赖方向。本规范由 ArchUnit 机械化守护。
> **状态**: 强制 · 违反即阻断 CI

## 2. 模块职责与依赖方向（ADR-002）

| 模块 | 职责 | 允许依赖 |
|------|------|---------|
| `huazai-trip-common` | model / dto / enums / exception / util / constant | 无（仅 JDK + 基础库） |
| `huazai-trip-agent-*` | 单个 Agent 的核心逻辑、Tool、Service、Repository | common, skills, agentscope, spring |
| `huazai-trip-skills` | 业务 Skills（跨 Agent 复用） | common, agentscope（**禁止依赖任何 agent**） |
| `huazai-trip-server` | Web 服务、门面、配置 | common, agents, skills, spring |
| `huazai-trip-tests` | E2E / 集成 / 架构约束 | 全部（仅测试作用域） |

**依赖方向（单向，不可逆）**:

```
common  ←  skills  ←  agents  ←  server  ←  tests
   ↑__________↑__________↑_________↑
       （上游永不依赖下游）
```

### 2.1 Agent 设计 5 原则
1. **单一职责** —— 一个 Agent 只做一件事，做好一件事
2. **显式通信** —— Agent 间只通过 A2A 协议（`Msg`）交互，禁直接方法调用
3. **可观测性** —— 每个 Agent 暴露健康检查 + 关键指标
4. **可中断性** —— 长任务支持暂停/恢复/取消
5. **无状态** —— Agent 自身无状态，状态存于 Redis / ReMe

### 2.3 Server 与 Agent 边界
- `huazai-trip-server` 负责 REST 入口、Facade、会话协调、结果装配，不承载具体 Agent 业务决策。
- `Controller` 只能依赖 `Facade / Application Service`，不得直接依赖具体 `*Agent` 类。
- `Facade / Application Service` 可以编排 Agent，但不得在 server 层复制 Route / Itinerary / Budget 等核心业务规则。
- 需要跨多个 Agent 的业务编排时，优先下沉到 `SupervisorAgent` 或专用编排层，而不是堆在 server。
- `server` 层若开始出现多 Agent 规则拼装、重试/降级复制、领域规则判断，视为分层漂移，必须拆回 agent 或 shared service。

### 2.2 Agent 唯一标识与核心能力
| Agent | 职责 | A2A 标识 | 核心业务 Skill |
|-------|------|----------|---------------|
| SupervisorAgent | 需求理解、任务分解、结果汇总 | `agent://supervisor` | plan-notebook |
| XHSAnalysisAgent | 小红书笔记检索与分析 | `agent://xhs-analyst` | xhs-note-analysis |
| RouteAgent | 景点间交通路线规划 | `agent://route-planner` | route-planning |
| ItineraryAgent | 每日行程编排 | `agent://itinerary-designer` | itinerary-design |
| BudgetAgent | 费用计算与预算管控 | `agent://budget-controller` | cost-calculation |

## 3. 架构铁律（ArchUnit 守护）

| 规则 | 描述 |
|------|------|
| R1 | **Agent 之间禁止直接依赖** —— 只能通过 A2A 协议通信 |
| R2 | **Skill 禁止依赖任何 Agent** —— Skill 是无状态可复用单元 |
| R3 | **Controller 禁止直接调用 Agent** —— 必须经 Service/Facade |
| R4 | **common 禁止依赖任何业务模块** —— 它是最底层基座 |
| R5 | **禁止跨 Agent 引用内部包**（`..internal..`、`..impl..` 私有类） |
| R6 | **Server 禁止复制 Agent 业务规则** —— 仅做入口/门面/编排，不重复实现核心规划逻辑 |

```java
@AnalyzeClasses(packages = "com.nanobot")
public class ArchitectureConstraints {

    @ArchTest
    static final ArchRule agents_must_not_depend_on_each_other =
        classes().that().resideInAPackage("..agent..")
            .should().onlyDependOnClassesThat()
            .resideInAnyPackage("..common..", "..skills..",
                "io.agentscope..", "java..", "org.springframework..");

    @ArchTest
    static final ArchRule skills_must_not_depend_on_agents =
        noClasses().that().resideInAPackage("..skills..")
            .should().dependOnClassesThat().resideInAPackage("..agent..");

    @ArchTest
    static final ArchRule controller_must_not_call_agents =
        noClasses().that().resideInAPackage("..controller..")
            .should().dependOnClassesThat().resideInAPackage("..agent..");
}
```

## 4. 包结构规范（模块内部）

```
com.nanobot.<module>/
├── <XxxAgent>.java        # Agent 入口（仅 agent-* 模块）
├── config/                # Spring 配置
├── tool/                  # LLM 可调用的 Tool 封装
├── mcp/                   # MCP 协议适配（仅 route 模块）
├── service/               # 业务服务
├── repository/            # 数据访问（Redis / 缓存）
├── model/                 # 模块私有模型（公共模型放 common）
└── ...
```

**命名约定**:
- 包名全小写，单数（`tool` 不是 `tools`）
- Agent 入口类以 `Agent` 结尾
- Tool 类以 `Tool` 结尾，Service 以 `Service` 结尾

## 5. 文件组织约束

| 约束 | 规则 | 守护手段 |
|------|------|---------|
| 文件长度 | ≤ 500 行 | Checkstyle FileLength |
| 方法长度 | ≤ 50 行 | Checkstyle MethodLength |
| 圈复杂度 | ≤ 10 | Checkstyle CyclomaticComplexity |
| 一个文件一个 public 类 | 强制 | Checkstyle OuterTypeNumber |
| 禁止 SNAPSHOT 依赖 | 强制 | Maven Enforcer |

## 6. 新增模块/feature 的目录约定

- 新 Agent → 新建 `huazai-trip-agent-<name>/`，并在根 POM `<modules>` 注册
- 新业务能力 → 在 `huazai-trip-skills/` 下按领域分包新增 Skill
- 新需求 → 在 `.harness/changes/<id>/` 建档（见开发流程规范）
- 新架构决策 → 追加 ADR（外部 `specs/architecture.md` 或本体系约定位置）
